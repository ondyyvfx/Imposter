# 🕵️ Шпион — игра с друзьями (Next.js + Supabase)

Веб-платформа для игры в «Шпиона» (импостера): создаёшь комнату, друзья заходят
по коду, ведущий запускает игру — система **сама случайно выбирает шпиона** и
раздаёт остальным **одно и то же загаданное слово**. Шпион слова не знает —
получает только **далёкую подсказку**, чтобы пару кругов продержаться и не
спалиться сразу.

У каждого игрока появляется **карточка**: нажимаешь — она переворачивается. Если
ты в команде — увидишь слово. Если шпион — увидишь, что ты шпион, и подсказку.

- ✅ Комнаты по коду + realtime-лобби
- ✅ Случайный шпион и случайное слово при каждом старте
- ✅ Слова **не повторяются** в рамках одной комнаты
- ✅ 6 категорий: Аниме, Фильмы, Музыка, Места, Страны, Мемы (~130 слов)
- ✅ Честно: слово шпиону **не отправляется** на устройство (раздаётся персонально с сервера)
- ✅ Современный минималистичный дизайн, адаптив под мобилки
- ✅ Кнопка «Раскрыть шпиона» и «Новый раунд» для ведущего

## 1. Настройка базы (Supabase)

1. Зайди на [supabase.com](https://supabase.com) → **New project** (запиши пароль БД).
2. В проекте открой **SQL Editor → New query**.
3. Открой файл [`supabase/dump.sql`](./supabase/dump.sql), скопируй **всё**
   содержимое, вставь и нажми **Run**.
   - Создаст таблицы, RLS-политики, включит Realtime и зальёт все слова.
   - Скрипт можно запускать повторно — он сам пересоздаёт таблицы.
4. Открой **Project Settings → API** и скопируй три значения:
   - `Project URL`
   - `anon` `public` ключ
   - `service_role` `secret` ключ ⚠️ (держи в секрете, только для сервера)

## 2. Деплой на Vercel (самый простой путь)

1. Залей проект в GitHub (см. раздел ниже) и зайди на
   [vercel.com](https://vercel.com) → **Add New → Project** → выбери репозиторий.
2. Vercel сам определит, что это Next.js — ничего настраивать не нужно.
3. В разделе **Environment Variables** добавь три переменные:

   | Name | Value |
   | --- | --- |
   | `NEXT_PUBLIC_SUPABASE_URL` | Project URL из Supabase |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | anon public ключ |
   | `SUPABASE_SERVICE_ROLE_KEY` | service_role secret ключ |

4. Нажми **Deploy**. Через минуту получишь ссылку вида
   `https://твой-проект.vercel.app` — кидай её друзьям.

> Если меняешь переменные окружения уже после деплоя — нажми **Redeploy**.

## Как играть

1. Хост жмёт **«Создать комнату»**, вводит имя — получает код (например `KQ7MN`).
2. Друзья открывают сайт → **«Войти по коду»** → вводят имя и код. Или просто
   переходят по ссылке `…/room/КОД`.
3. Все собрались в лобби — хост выбирает **категорию** и жмёт **«Начать игру»**.
4. Система случайно назначает шпиона и выдаёт слово остальным. У каждого —
   карточка: тапни, чтобы открыть свою роль.
5. По очереди задавайте вопросы / описывайте слово, не называя его. Вычисляйте,
   кто шпион.
6. Хост жмёт **«Раскрыть шпиона»** — всем покажет, кто это был и какое слово.
7. **«Новый раунд»** — новый случайный шпион и новое слово (уже загаданные в этой
   комнате не повторятся).

## Как это устроено (кратко)

- **Next.js (App Router)** — фронтенд + API-роуты (`src/app/api/...`).
- **Supabase (Postgres)** — хранит комнаты, игроков, слова и раунды.
- Секретные данные раунда (слово и кто шпион) лежат в таблице `rounds`, закрытой
  через RLS. Браузер их напрямую прочитать не может. Карточка отдаётся **персонально**
  через `GET /api/rooms/[code]/card`: шпиону уходит только подсказка, команде — слово.
- Обновления лобби идут через **Supabase Realtime** + резервный поллинг каждые
  2.5 c (работает, даже если realtime отключён).

### Структура

```
supabase/dump.sql          ← дамп БД (схема + RLS + realtime + слова)
src/app/page.tsx           ← главная (создать / войти)
src/app/room/[code]/       ← комната: лобби и игра
src/app/api/rooms/...      ← вся серверная логика игры
src/lib/                   ← клиенты Supabase, типы, утилиты
```

### Хочешь добавить свои слова?

Просто вставь строки в таблицу `words` (через Supabase Table Editor или SQL):

```sql
insert into public.words (category, word, hint) values
  ('Мемы', 'Твоё слово', 'Очень далёкая подсказка для шпиона');
```

Новая категория появится в выборе автоматически.
